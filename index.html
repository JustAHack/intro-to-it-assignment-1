Project

Overview

Discord is a widely used social media and communication tool loved and used by millions of people around the world. However, what many users do not know is that Discord tracks every small detail about its users and what they are doing on the platform. What I propose to build is a similar product that provides many of the same features over time, but including other features that include far greater privacy for each user. Including but not limited to text, audio and video communication. Ability to link to and communicate with other types of media such as other websites, games, software etc. Plugins built by users for games that can communicate back to this software providing information such as new loot obtained, levels gained, achievements gained etc.

Motivation

Privacy is a major concern for everyone in this digital age. Peoples personal data is being sold without their knowledge because either people do not read the terms and services or sometimes companies don't even let their users know what is happening with the data they obtain. This can lead to information being stolen or used for nefarious reasons after it has been sold to a third party. My goal is to create a service that people love to use, communicate and have fun with their friends, families and peers while being assured that their data is their own and not be sold to any third party. This would be achievable with technology available today.

Description

Discord provides a fantastic platform for its target audience but it has its pitfalls in terms of privacy. Discord supposedly does not encrypt user information on their servers, everything sits there ready to be accessed at any moment. It is not so much a matter of if, but when discord has a breach and data is stolen as we have seen with many other companies, including facebook among others. It is my plan to build a more decentralised social media platform that revolves around games as its main focus while providing a place for communities, friends and peers to communicate freely and without tracking. This could be achieved through the use of a project known as Matrix and some proper planning and design. Users should be able to host their own servers where people can join and communicate through text, audio and video as the main feature. Users should also be able to link their profiles to the platform and have privacy settings attached to their profiles so that they can feel comfortable to share only as much information as they are happy to share publicly or privately. The platform should provide users the ability to create plugins by using an API to communicate data between their servers, profile etc and games. (E.g. achieving a new level, obtaining new gear, planning raids etc). Communities should be able to link third party social media and software to their servers through API. Communities can potentially create their own styling for their servers to fit their own themes to make a more engaging and rich experience. Authentication and authorisation to be built in so that accounts are secure. These are the bare minimum features for the product to succeed and a prototype to be built with. Users should be able to delete their messages on any server, personal chat room etc and that data be permanently removed from all devices that message was sent to that is connected to currently and in future as to assure data is not permanently tracked and kept.

Tools and technologies required

HTML5, CSS3 and JavaScript for building the frontend of the software, so that a rich user experience can be provided and users can use the software on the web as well as on computer or mobile devices. A serverside language to help support the features and requirements of the project such as NodeJS, C#, Java etc. Matrix from Matrix.org to provide the main functionality described above such as end-to-end encrypted decentralised communications. Ability to communicate with third party programs such as Discord, Facebook WhatsApp, IRC, Mumble, LinkedIn Messaging, Twitter etc. Figma for designing and prototyping frontend user interface and user experience. Git and GitHub for managing and tracking the code and relevant import files for the project. Amazon Web Services (AWS) or Microsoft Azure for infrastructure hosting such as Amazons EC2 for hosting, Amazon S3 for cloud storage of images, Amazon DynamoDB for bare minimum data requirements (authentication) Trello for helping to manage requirements and progress on the project and direct / track tasks. A code editor (e.g. VS Code, Atom) to write code to build the underlying infrastructure.
